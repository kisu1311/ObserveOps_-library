system.cpu.percent

Lower is better

Description

Percentage of total CPU time in use across all cores, from 0% (idle) to 100% (fully utilised). Used for load monitoring, alerting and capacity planning.

Interpretation

  • High values: above 80% indicates CPU saturation and potential performance issues.

  • Low values: below 5% means the CPU is mostly idle — normal during off-peak windows.

RCA: when high, correlate with related counters, recent config changes and active alerts on the same source.

system.memory.free.percent

Higher is better

Description

Percentage of physical memory currently free and available for allocation.

Interpretation

  • High values: plenty of headroom for new allocations.

  • Low values: can trigger swapping and degraded performance.

RCA: when low, check for memory leaks, oversized caches and recent deploys on the host.

The dashboard's AI Insights feed renderer. A declarative mount (data-widget="aiinsights") is filled by the OOW.aiinsights renderer with a role="list" column of compact insight cards. Each card carries a coloured severity rail on its left edge, an uppercase "kind" label (anomaly / forecast / root cause / noise) tinted by its own category colour, a right-aligned mono confidence read-out, a bold title and a muted one-line description. The renderer emits structure only; the severity rail (--ins-sev) and kind tint (--ins-kc) are injected inline as var(--token) references, so aiinsights.css owns all presentation and nothing hardcodes palette hex. Confidence percentages are the only seed-driven data, so output is deterministic and renders identically every load. Rows lift on hover, expose a keyboard focus ring when made interactive for drill-down (rendered as button/link), and an aria-disabled feed dims the whole set for no-data / loading. Depends on the shared rng / esc utils with small local fallbacks.

Anomaly Detection Widgetwidgets/anomaly
The dashboard's AI/ML anomaly-detection renderer. A declarative mount (data-widget="anomaly") is filled by the OOW.anomaly renderer with a single token-themed SVG: a seeded time-series drawn as a smooth cubic-bezier line, wrapped in a shaded confidence band (the learned baseline ± a tolerance), a faint dashed mean baseline, and any sample that breaches the band flagged as a red dot inside a soft pulsing ring. The series colour is data-driven (data-color, a SVAR key or hex) and injected as generated SVG data, while the band opacity, baseline and the always-red anomaly markers are decoupled from the series colour and themed from design tokens. Output is deterministic from opts.seed, so it renders identically every load. Depends on the shared series / smoothPath / rng / SVAR utils. Presentational by default, with a focus ring when the mount is focusable and a stale/paused disabled treatment.

Bottom-center transient toast for action confirmations. Refined from the live single .toast pill (one node the global toast() function rewrote and flashed via a .show class) into a reusable, semantic notification: an icon chip, a message with an optional dismiss (✕), and one local --oo-toast-accent custom property so a tone modifier (success / info / error / warning) only flips the accent + tint. A separate .oo-toast-host fixes a stack of toasts to the viewport bottom-center, or with --contained confines them to a positioned ancestor (the gallery stage). Resting toasts stay fully visible so they read statically; spawned ones run the .is-show enter animation and fade out with .is-hiding, mirroring the live 2.4s auto-dismiss. Honours prefers-reduced-motion; close button carries hover / focus-visible / disabled states.

Fullscreen widget modal: a deep, blurred backdrop scrim centring one large card (header bar with widget title + teal type tag + ✕ close, over a big scrollable body).
