App shell — the frame every module reuses
Icon rail
Secondary list / tree
monitors · apps · settings
icon + title · ☆24h ▾

Top bar (metadata wordmark · global search · health/diagnostics icons · notification bell with a red count · BUILD pill · user avatar) sits above a collapsible 44px icon rail of 16 modules (active = teal). Each module opens to a page header (module icon + title; dashboards add a ☆ favourite & a collapse chevron) and a content area. Sub-navigation is either a row of tabs or, in Settings, a searchable left list. Only the content changes between modules — the chrome is identical.

Core interaction patterns

◷Time range & filtering

Top-right on every data screen: a relative pill (5m · 1h · 24h · 10d · Today · This Month) + the resolved absolute range, a refresh interval (5m) and a Custom range. Dashboards add a draggable timeline brush above the grid.

Do: pick a sensible default range and always show the resolved absolute window.

⤓Toolbar actions

A consistent right-aligned cluster: chart-type switch, snapshot / image, export, share, Save View, fullscreen, and a module “Create … View”.

Do: keep the primary action (Save View / Create) at the far right of the bar.

▦Dashboards & widgets

A widget grid of KPI tiles + SVG charts: donut with a centre total, stacked bars, a stacked time-series trend, and radial gauges.

Do: one metric per tile; numbers in JetBrains Mono tabular-nums; severity color only on the value or status dot.

◉Severity summary chips

Modules headline their state with count chips — Alerts shows Down · Unreachable · Critical · Major · Warning with live counts.

Do: order chips by severity, color the count, and always keep the text label.

≣Data tables & lists

Ant Design tables — compact 34–36px rows, 1px dividers, sticky header, a checkbox bulk column, a ⋯ row menu, a list / grid toggle and pagination.

Do: lead each row with a status signal. Don’t exceed ~7 columns — push the rest into the drawer.

⌗Explorer / query builder

Explorers pair a left builder panel (metric tabs, “Select Monitor”, “+”) with a chart canvas; drag a metric onto a slot to plot it.

Do: show “Drop metric here to view trend” on empty slots so the canvas is never blank.

▤Detail drawer & modals

A right-side drawer inspects a record in context (Esc / click-away closes); modals handle focused create / edit / confirm; dropdowns carry row & toolbar actions.

Do: drawer to inspect-in-context, modal for blocking tasks, and always confirm destructive actions.

⌁Empty · loading · live

Centred empty states (icon + one-line message + CTA), skeleton shimmer while loading, and a live mode (Log Explorer’s Start Live Trail). Beta features carry a BETA badge.

Do: never leave a bare blank panel — show a skeleton, empty, or error state.

Down#c84235
Unreachable#8c5bd8
Major#f47c22
Warning#e8b407

Each severity is a token family — --severity-{state} with -lighter / -lightest tints (chip & row backgrounds), -darker (text) and -dot-box (the status dot). It is reserved for state — as a dot, tag/pill, badge count (the ● on the bell) or row tint — never for decoration or branding.

Quick do & don’t
✔ Do
  • Reuse the app-shell chrome & the spacing scale on every screen.
  • Pair every severity color with text or an icon (accessibility).
  • Use mono tabular figures for metrics, IDs & timestamps.
  • Keep the primary action at the top-right of the toolbar.
✘ Don’t
  • Invent accent or status colors outside the token families.
  • Use severity red for emphasis or branding.
  • Crowd tables — push detail into the drawer.
  • Show relative-only or absolute-only time; show both.
